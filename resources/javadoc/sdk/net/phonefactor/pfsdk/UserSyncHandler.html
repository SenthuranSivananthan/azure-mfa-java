Class UserSyncHandler

public class UserSyncHandler
extends java.lang.Object
The UserSyncHandler is a class that is fed to the userSync() method of the PFAuth class as a way to handle sync events received from the PhoneFactor servers. You should derive from this class and implement any sync methods you wish to handle. When you override a method, returning false from your implementation will cause the given event to fail. Returning true will signal the PhoneFactor servers that sync succeeded. It is suggested that you not commit your work in the handler methods you implement because future parsing or events could cause failure and no response to be sent back to the PhoneFactor servers. A response to the PhoneFactor servers is sent AFTER all calls to your handler methods have been completed. Ideally, you should commit your work after the PFAuth userSync() method returns -- at that point, communication with the PhoneFactor servers has been done and the results of your handler calls has been communicated and accepted.

onChangedInfo

public boolean onChangedInfo(UserSyncHandler.ChangedInfo i)
            throws UserSyncHandler.ChangedInfoSkippedException
Handler for when user information has changed on the PhoneFactor servers. Your implementing class should return true to indicate you have processed and accepted the change or false to indicate that you did not (and will not) accept the change. Returning true/false will cause the pending change to be removed from the PhoneFactor servers -- if you want to skip/ignore a given change, throw a ChangedInfoSkippedException. Throwing the ChangedInfoSkippedException will not abort the sync process -- sync will continue with the next item from the server and the skipped item will remain on the PhoneFactor servers. You should skip a change when you cannot process it now, but may be able to process it later, and you do not want to throw away the change (for instance: The referenced user is not found in your database, but it may be found later.)
Parameters:
i - ChangedInfo The information about a given user that has changed.
Returns:
True/false depending on if your handler code accepted the change.
Throws:
UserSyncHandler.ChangedInfoSkippedException - Throw this to skip handling a given ChangedInfo for now.
