net.phonefactor.pfsdk

Class PFAuth

  • java.lang.Object
    • net.phonefactor.pfsdk.PFAuth
public class PFAuth
extends java.lang.Object
The PFAuth class is the core utility object for authenticating via the PhoneFactor SDK. You should create, initialize, and use one instance of this object across all threads in your application. The object is thread safe across calls to authenticate() and userSync() after you finish initializing the object. It is suggested that initialization be done at application startup since initialization requires reading files from disk. It will not hurt to create, setup, and use as many instances as you want, but each instance will need to be initialized.

authenticate

public PFAuthResult authenticate(PFAuthParams authParams)
            throws SecurityException,
                   XMLException,
                   InterruptedException,
                   TimeoutException,
                   CommunicationException,
                   ParseException,
                   MalformedMessageException
Authenticates a user via PhoneFactor. You will always get a valid PFAuthResult or an exception will be thrown.
Parameters:
authParams - The parameters to use for the authentication See PFAuthParams for detailed information
Returns:
A PFAuthResult instance that contains the results of the authentication request
Throws:
SecurityException - Thrown when a security issue prevents success (ex: Remote certificate is valid but isn't for a PhoneFactor server)
XMLException - Thrown when the underlying xml routines fail in a non recoverable way
InterruptedException - Thrown when the current thread is interrupted and we quit waiting on the PhoneFactor server
TimeoutException - Thrown when no response was received from the PhoneFactor server after 60 seconds
CommunicationException - Thrown when communication with the PhoneFactor server breaks down
ParseException - Thrown when the response from the PhoneFactor server could not be parsed
MalformedMessageException - Thrown when a response message is received from the PhoneFactor server and parsed correctly but is in the wrong format
